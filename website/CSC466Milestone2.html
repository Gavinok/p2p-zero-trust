CSC466 Project: Milestone 2

For this milestone, we committed to have a basic peer-to-peer network developed locally to serve as a basis for project development and testing using Docker to facilitate simulation of a realistic peer-to-peer environment. We are pleased to report that the peer-to-peer docker network has been implemented and is functioning as a testing and development environment for future work on this project. Using docker-compose, a toolset developed by Docker, we were able to create two simulated machines that act as peers in our network. We have created two services, one for each peer, to act as a basic system to conduct zero trust network implementation and send messages through simple TCP sockets.

Technically speaking, we have one sender peer and one receiver peer. The receiver peer listens on port 9000 and (dangerously) accepts any attempt to connect. The purpose of our project is to implement zero trust properties to secure messages to the receiver peer. The sender peer sends a message, "hello", every 3 seconds. Currently, these messages are not encrypted in any way and are simply sent as plaintext over the network, so the system is vulnerable to threats such as packet interception and man-in-the-middle attacks.

The current state of the network is certainly high-risk, but is a basis for us to build our zero trust functionality on top of. We have metrics for measuring latency of messages sent between the peers, and plan on reporting noticeable changes in performance as we incrementally implement the zero trust network, reevaluating at each step if performance is affected by the implemented features.

One of our goals for Milestone 3 are to implement the top priority features of a zero trust network, as follows:

  • All network flow MUST be authenticated before being processed
  • Authentication and encryption MUST be performed by the the application-layer endpoints
  • All network flow MUST be enumerated so that access can be enforced by the system

In addition, we plan on improving our performance testing metrics to further evaluate the impact of zero trust network features on a peer-to-peer network.
